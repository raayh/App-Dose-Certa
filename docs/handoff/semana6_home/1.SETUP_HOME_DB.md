# 📊 Handoff: Modelagem de Medicamentos (Home)

Este guia é para o Engenheiro de Dados. Seu objetivo é estruturar o modelo de dados dos remédios que serão exibidos na Home e salvos na tela de cadastro.

---

## 🏗️ Passo 1: Adicionar a Interface de Medicamentos

No arquivo `types/database.ts` (que já deve conter a interface de `User`), adicione o seguinte modelo:

```typescript
// ---------------------------------------------------------
// MEDICAMENTOS (Coleção "medications")
// Usamos arrays para horários e sintomas para evitar tabelas extras.
// ---------------------------------------------------------
export interface Medication {
  id?: string;           // Gerado automaticamente pelo Firebase
  user_id: string;       // ID do dono do remédio (Relacionamento)
  name: string;          // Nome do remédio
  format: string;        // Ex: "Comprimido", "Gotas", "Cápsula"
  dosage: string;        // Ex: "500mg" ou "2ml"
  times_per_day: number; // Frequência diária
  duration_days: number; // Quantos dias vai durar o tratamento
  amount_per_time: number; // Quantos comprimidos/gotas por dose
  initial_date: Date;    // Quando começa o tratamento
  created_at: Date;      // Data do cadastro
  times: string[];       // Array de horários (Ex: ["08:00", "20:00"])
  symptoms: string[];    // Sintomas relatados (Opcional por enquanto)
}
```

---

## 💡 Dicas de Implementação

1. **Desnormalização:** No NoSQL, preferimos salvar tudo dentro do mesmo documento em vez de criar várias tabelas. Por isso usamos `string[]` para os horários.
2. **Relacionamento:** O campo `user_id` é o que garante que eu só veja os meus remédios e você veja os seus. Nunca esqueça de filtrar por esse campo nas consultas!
