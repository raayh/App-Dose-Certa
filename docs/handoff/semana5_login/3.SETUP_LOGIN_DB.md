# **🗃️ SETUP LOGIN: BANCO DE DADOS (Firestore)**

Este guia é para o Engenheiro de Dados. Seu objetivo é estruturar o "Schema" do banco NoSQL usando TypeScript, e atrelar a primeira gravação na Nuvem (Usuários) à criação de conta.

---

## 📊 Passo 1: Construir a Estrutura (Interfaces)

Sendo o Firebase um banco Schemaless (que aceita qualquer formato), precisamos cravar regras estritas no código. 

Crie este arquivo com os dois modelos principais do projeto:
**Arquivo:** `types/database.ts`
```typescript
// ==========================================
// 📊 MODELAGEM DE DADOS DO FIREBASE (NOSQL)
// ==========================================

// ---------------------------------------------------------
// 1. USUÁRIOS (Coleção "users")
// Só precisamos salvar informações extras aqui. Senhas NUNCA vêm para cá.
// ---------------------------------------------------------
export interface User {
  id: string; // Chave primária gerada pelo auth
  email: string;
  created_at: Date;
}
```

---

## 🔌 Passo 2: O Gancho de Criação (SetDoc)

Quando o módulo de autenticação gera um usuário, o Banco precisa tomar conhecimento dele. 

No arquivo de Login, adicione as importações do Firestore no cabeçalho:
**Arquivo:** `app/(auth)/login.tsx`
```tsx
import { doc, setDoc } from 'firebase/firestore';
import { db } from '@/services/firebaseConfig'; // Puxa do backend
```

Desça até a função `handleSignUp` e embuta o código de escrita imediatamente **após** criar a conta e **antes** de enviar o e-mail:
```tsx
  // ... trecho original
  const credentials = await createUserWithEmailAndPassword(auth, email, password);
  const user = credentials.user;

  // =========================================
  // BANCO DE DADOS (Firestore)
  // 1. doc(): Localiza/Cria a "gaveta" 'users' usando a chave secreta 'user.uid'
  // 2. setDoc(): Guarda os dados lá dentro
  // =========================================
  await setDoc(doc(db, "users", user.uid), {
    email: user.email,
    created_at: new Date()
  });

  await sendEmailVerification(user); 
  // ... resto do código original
```
