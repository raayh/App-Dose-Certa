# Guia de Implementação Técnica: Tela de Login 🔐

Este guia detalha como construímos a tela de login, desde a estrutura visual até a lógica de conexão com o Firebase.

---

## 🎨 Fase 1: Interface (O "Corpo" do App)

O primeiro passo foi criar a estrutura visual usando o sistema de **Views** (contêineres).

### 1. Estrutura de Contêineres
Dividimos a tela em 3 partes fundamentais para facilitar o alinhamento.

**Arquivo:** `app/(auth)/login.tsx`
```tsx
import React from 'react';
import { View, Text, StyleSheet, Image, TouchableOpacity, TextInput } from 'react-native';

export default function LoginScreen() {
  return (
    <View style={styles.container}>
      {/* Parte 1: Topo */}
      <View style={styles.header}>
        <View style={styles.header1}> 
            <Image source={require('@/assets/images/logo.png')} style={styles.logo} />
            <Text style={styles.headerTitle}>Dose Certa</Text>
        </View>
      </View>

      {/* Parte 2: Formulários */}
      <View style={styles.formContainer}>
        <Text style={styles.label}>E-mail</Text>
        <TextInput style={styles.input} placeholder="seu@email.com" />
      </View>

      {/* Parte 3: Botão */}
      <TouchableOpacity style={styles.button}>
        <Text style={styles.buttonText}>Entrar</Text>
      </TouchableOpacity>
    </View>
  );
}
```

> [!TIP]
> **Explicação do Coach:** Usamos `flexDirection: 'row'` no `header1` para colocar a imagem e o texto um ao lado do outro. No React Native, o padrão é um embaixo do outro (`column`).

## ⚙️ Fase 2: Lógica e Autenticação (A "Alma" do App)

Agora que a tela existe, precisamos de memória (`state`) e ação (`functions`).

### 1. Criando os Estados
Importamos o `useState` para que o app lembre do que o usuário digita.

```tsx
const [email, setEmail] = useState('');
const [password, setPassword] = useState('');
const [loading, setLoading] = useState(false);
```

### 2. A Função de Login (Firebase)
Usamos uma função assíncrona porque o Firebase demora alguns segundos para validar os dados na nuvem.

```tsx
const handleLogin = async () => {
  setLoading(true); // Ativa o sinal de carregando
  try {
    await signInWithEmailAndPassword(auth, email, password);
    // Se der certo, o app muda de tela automaticamente via _layout.tsx
  } catch (error) {
    Alert.alert("Erro de Login", "Verifique seu e-mail e senha.");
  } finally {
    setLoading(false); // Desativa o carregando sempre
  }
};
```

### 3. Conectando tudo no JSX
Nesta etapa final, ligamos as variáveis aos componentes:

```tsx
<TextInput 
  value={email}
  onChangeText={setEmail}
  secureTextEntry={false}
/>

<TouchableOpacity onPress={handleLogin} disabled={loading}>
  {loading ? <ActivityIndicator /> : <Text>Entrar</Text>}
</TouchableOpacity>
```

## 🔒 Fase 3: Segurança e Persistência

Para garantir que o aplicativo funcione como em empresas reais (tipo Netflix ou bancos), implementamos 3 camadas:

### 1. Memória de Longo Prazo (AsyncStorage)
Configuramos o Firebase no `services/firebaseConfig.ts` para salvar a sessão no celular do usuário:
```ts
export const auth = initializeAuth(app, {
  persistence: getReactNativePersistence(ReactNativeAsyncStorage)
});
```

### 2. O Vigilante de Rotas (Auth Guard)
No arquivo `app/_layout.tsx`, ficamos "escutando" as mudanças do Firebase. Se existir um usuário, o Vigilante teleporta ele para a Home. Se não existir, trava ele no Login.

### 3. Verificação de E-mail (Email Verification)
Impedimos que contas falsas adentrem o sistema.
No Login: Checamos se `credenciais.user.emailVerified` é verdadeiro. Senão, aplicamos `signOut()`.
No Cadastro: Após criar, ativamos `sendEmailVerification(user)` e desconectamos o usuário para forçar o clique no e-mail.

## 🗃️ Fase 4: Integração Básica com Banco (Firestore)

Aproveitando a tela de Cadastro, criamos a nossa primeira persistência de dados.

Quando chamamos `createUserWithEmailAndPassword`, nós atrelamos a criação da gaveta no banco de dados usando o módulo `doc` e `setDoc`:

```tsx
// Salva quem é o usuário recém-criado na coleção 'users'
await setDoc(doc(db, "users", user.uid), {
  email: user.email,
  created_at: new Date()
});
```
