# **⚙️ SETUP HISTÓRICO: BACK-END & INTEGRAÇÃO (REGRAS DE NEGÓCIO)**

Este guia é focado no Desenvolvedor Integrador / Back-End. Ele aborda a construção do motor de cruzamento de dados de alta performance (JOIN em memória) e a lógica matemática dinâmica para alimentar o Banner de progresso diário.

---

## ⚡ 1. Os 5 Motores Inteligentes do Nosso Histórico

Diferente de telas comuns, o Histórico usa **arquitetura reativa avançada** por meio de 5 mecanismos principais executados ao carregar a tela:

### A. Busca Paralela (`Promise.all`)
Evita o gargalo ("waterfall") de carregar as receitas e depois o histórico. Carregamos ambas as queries ao mesmo tempo para reduzir o tempo de resposta da tela pela metade.

### B. Dicionário de Receitas O(1) (`medMap`)
Convertemos o array de remédios em uma tabela "chave-valor" indexada pelo ID do remédio. Isso possibilita cruzar dados de milhares de registros sem sofrer lentidão algorítmica.

### C. Escudo de Integridade de Dados (`medInfo?.type || 'outros'`)
Garante que se uma receita original for deletada no futuro, o app exibe um fallback genérico no card em vez de travar com tela branca de erro.

### D. Motor Matemático da Contagem Esperada (`countExpected`)
Não confia apenas nos registros de histórico salvos. O motor varre ativamente o cadastro do usuário filtrando o que está planejado para HOJE (considerando `startDate`, `endDate` e a quantidade de horários no array `times`) para calcular o divisor real do progresso (ex: "X de **Y**").

### E. Algoritmo de Agrupamento Cronológico em Memória (`sections`)
Transforma uma lista plana de registros do banco em uma estrutura complexa de Grupos com as chaves dinâmicas `"Hoje"`, `"Ontem"` ou a data formatada em português para o `SectionList`.

## 🚀 2. O Código Completo de Integração (Backend Real)

Substitua a sua função de busca e lógica superior no componente real `app/(tabs)/explore.tsx` pelo motor completo e tipado abaixo.

```tsx
// 1. IMPORTS DO FIREBASE E TIPOS
import React, { useState, useEffect } from 'react';
import { auth, db } from '@/services/firebaseConfig';
import { MedicationHistory } from '@/types/database';
import { collection, query, where, getDocs, orderBy } from 'firebase/firestore';

// ... (medicationImages e imports visuais permanecem do Front)

export default function HistoryScreen() {
  const user = auth.currentUser;
  
  // 2. ESTADOS TIPADOS E MEMÓRIAS
  const [history, setHistory] = useState<{ 
    title: string; 
    data: (MedicationHistory & { type: string, dosage: string })[] 
  }[]>([]);
  const [totalRealToday, setTotalRealToday] = useState(0);
  const [loading, setLoading] = useState(true);

  // 3. O MOTOR PRINCIPAL DA BUSCA E PROCESSAMENTO
  async function fetchHistory() {
    if (!user) return;

    try {
      // QUERY A: Pesquisa todas as receitas do usuário (Para o dicionário de ícones e soma real)
      const medQuery = query(
        collection(db, "medications"),
        where("user_id", "==", user.uid)
      );

      // QUERY B: Pesquisa o histórico cronológico ordenado (Necessita do Índice Composto!)
      const histQuery = query(
        collection(db, "history"),
        where("user_id", "==", user.uid),
        orderBy("taken_at", "desc")
      );

      // EXECUÇÃO SIMULTÂNEA DE ALTA VELOCIDADE
      const [medSnapshot, histSnapshot] = await Promise.all([
        getDocs(medQuery),
        getDocs(histQuery)
      ]);

      // A. GERANDO DICIONÁRIO DE BUSCA INSTANTÂNEA O(1)
      const medMap: Record<string, any> = {};
      medSnapshot.docs.forEach(doc => {
        medMap[doc.id] = doc.data(); 
      });

      // B. 🧮 MOTOR MATEMÁTICO: SOMA DE DOSES PLANEJADAS PARA HOJE
      const todayStr = new Date().toISOString().split('T')[0];
      let countExpected = 0;

      medSnapshot.docs.forEach(doc => {
        const med = doc.data();
        const isStarted = todayStr >= med.startDate;
        const isNotFinished = !med.endDate || todayStr <= med.endDate;

        if (isStarted && isNotFinished) {
          // Acumula o total de vezes que esse remédio toca no dia
          countExpected += med.times ? med.times.length : 0;
        }
      });
      setTotalRealToday(countExpected); 

      // C. CRUZAMENTO E ENRIQUECIMENTO DOS CARDS COM ESCUDO DE DELETADOS
      const richHistory = histSnapshot.docs.map(doc => {
        const histData = doc.data() as MedicationHistory; 
        const medInfo = medMap[histData.medication_id]; // O(1) Lookup!

        return {
          id: doc.id,
          ...histData,
          type: medInfo?.type || 'outros', // Fallback de segurança
          dosage: medInfo?.dosage || ''  // Fallback de segurança
        };
      });

      // D. ALGORITMO DE AGRUPAMENTO CRONOLÓGICO
      const groups: Record<string, any[]> = {};
      richHistory.forEach(item => {
        const dateKey = item.date; // "AAAA-MM-DD"
        if (!groups[dateKey]) {
          groups[dateKey] = [];
        }
        groups[dateKey].push(item);
      });

      // E. FORMATAÇÃO AMIGÁVEL DO SECTIONLIST
      const sections = Object.keys(groups).map(date => {
        const today = new Date().toISOString().split('T')[0];
        const yesterdayObj = new Date();
        yesterdayObj.setDate(yesterdayObj.getDate() - 1);
        const yesterday = yesterdayObj.toISOString().split('T')[0];

        let title = date;
        if (date === today) title = "Hoje";
        else if (date === yesterday) title = "Ontem";
        else {
          const [year, month, day] = date.split('-');
          title = `${day}/${month}/${year}`; // Formato BR
        }

        return {
          title: title,
          data: groups[date]
        };
      });

      setHistory(sections as any); 
      setLoading(false);

    } catch (error) {
      console.error("Erro no histórico:", error);
      setLoading(false);
    }
  }

  // 4. GATILHO AUTOMÁTICO DE ENTRADA NA TELA
  useEffect(() => {
    fetchHistory();
  }, []);

  // 5. ESCUDO E LÓGICA DO BANNER PREMIUM
  const todaySection = history.find((section: any) => section.title === "Hoje");
  const todayData = todaySection ? todaySection.data : [];

  const totalToday = totalRealToday; // Puxa do motor matemático real!
  const takenToday = todayData.filter((item: any) => item.taken).length; // Puxa do histórico filtrado

  // ... (A partir daqui segue o return do Front-End puro, utilizando totalToday e takenToday)
}
```
