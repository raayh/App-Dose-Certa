# **🗄️ SETUP HISTÓRICO: BANCO DE DADOS (FIRESTORE)**

Este guia é para o Desenvolvedor de Banco de Dados. A tela de Histórico utiliza duas coleções existentes (`medications` e `history`), mas exige a criação de um **Índice Composto** no Firestore para que a ordenação temporal segura funcione sem erros.

---

## 📊 1. Estrutura Lógica das Coleções Utilizadas

### A. Coleção `medications` (Leitura para Enriquecimento)
Utilizada como dicionário em memória para extrair os dados visuais que não salvamos no histórico (para evitar redundância).
*   **Chave Estrangeira:** `doc.id` (usado como `medication_id` no cruzamento).
*   **Campos Lidos:** `type` (determina o ícone) e `dosage` (exibido no card).

### B. Coleção `history` (Leitura Principal)
O motor de registros cronológicos das ações do usuário.
*   **Campos Filtrados:**
    *   `user_id` (Filtro de Segurança por usuário logado).
*   **Campos Ordenados:**
    *   `taken_at` (Filtro de Ordenação Cronológica Inversa - do mais novo para o mais antigo).

---

## 🚀 2. AÇÃO OBRIGATÓRIA: Configurar Índice Composto

No Firestore, por padrão, você não consegue **Filtrar por um campo** (`user_id`) e **Ordenar por outro** (`taken_at`) na mesma query sem criar um mapa de busca otimizado (Índice). Sem isso, a tela de Histórico falhará com o erro `FirebaseError: The query requires an index`.

### 🛠️ Como Criar o Índice no Console do Firebase:

1.  Acesse o console do seu projeto no Firebase.
2.  No menu lateral esquerdo, vá em **Build** -> **Firestore Database**.
3.  Clique na aba **Índices** (Indexes) no topo da tela.
4.  Clique no botão azul **Criar Índice** (Create Index) e preencha exatamente com estes dados:
    *   **ID da Coleção:** `history`
    *   **Campos para Indexar:**
        1.  `user_id` -> Tipo: **Ascendente**
        2.  `taken_at` -> Tipo: **Descendente**
    *   **Escopo da Consulta:** Coleção

5.  Clique em **Criar Índice** e aguarde de 2 a 3 minutos até o status mudar de *"Construindo"* para **"Ativo"**.

---

## 🛡️ Regras de Segurança (Security Rules)

Assegure-se de que as regras no arquivo `firestore.rules` permitem a leitura da coleção `history` apenas para o proprietário do dado:

```javascript
match /history/{documentId} {
  // Apenas o próprio usuário pode ler seu histórico
  allow read, write: if request.auth != null && request.auth.uid == resource.data.user_id;
  allow create: if request.auth != null && request.auth.uid == request.resource.data.user_id;
}
```
