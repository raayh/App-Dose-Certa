# **🗃️ SETUP HOME: BANCO DE DADOS (Firestore)**

Este guia é para o Engenheiro de Dados. Seu objetivo é estender o schema do banco para suportar o rastreamento das ações do usuário na Tela Home (Check-ins e Pulos de medicação).

---

## 📊 Passo 1: Modelagem do Histórico

A tela Home exige saber o que já foi tomado no dia para esconder os remédios da lista de pendentes. Para isso, criamos a coleção secundária `history`.

**Onde adicionar:** Abra o arquivo `types/database.ts` e cole a interface abaixo no **final do arquivo**.

**Arquivo:** `types/database.ts`
```typescript
// ---------------------------------------------------------
// 3. HISTÓRICO DE DOSES (Coleção "history")
// Armazena os registros de check-in para que a Home filtre o que
// ainda está pendente hoje.
// ---------------------------------------------------------
export interface MedicationHistory {
  id?: string;
  medication_id: string; // Chave Estrangeira do medicamento
  user_id: string; // Chave Estrangeira do usuário
  date: string; // Data da ação (Ex: "2026-05-13")
  time: string; // Horário da dose teórica (Ex: "08:00")
  status: 'taken' | 'skipped'; // Se o usuário tomou ou pulou
  taken_at: string; // Timestamp exato de quando o botão foi apertado
}
```

## 🔌 Passo 2: Vinculação de Segurança

A coleção `history` deve seguir as mesmas Regras de Segurança do Firebase:
- Leitura e Gravação permitidas APENAS onde o campo `user_id` é estritamente igual à identidade da sessão (`request.auth.uid`).

**IMPORTANTE:** O banco NÃO contém lógicas de filtragem, ele apenas armazena. O filtro MVP "Some da tela quando tomado" é responsabilidade estrita do Back-End em memória.
