## 👁️ Melhoria 1 — Mostrar/Ocultar Senha (Olhinho)

**Arquivo:** `app/(auth)/login.tsx`

### O que foi feito
Adicionado um botão de ícone dentro do campo de senha que alterna entre mostrar e ocultar os caracteres digitados.

### Estado criado
```tsx
const [showPassword, setShowPassword] = useState(false);

const showPasswordState = () => {
  setShowPassword(!showPassword); // inverte o valor atual
};
```

### Como funciona no campo
O campo usa `secureTextEntry` — quando `true`, os caracteres viram bolinhas. Ligamos isso ao estado:

```tsx
<View style={{ position: 'relative' }}>
  <TextInput
    style={styles.input}
    value={password}
    onChangeText={setPassword}
    secureTextEntry={!showPassword} // ← se showPassword=false, esconde. Se true, mostra.
    placeholder="123456"
  />

  {/* Ícone flutuante no canto direito do campo */}
  <TouchableOpacity
    onPress={showPasswordState}
    style={{ position: 'absolute', right: 15, top: 13 }}
  >
    <Ionicons
      name={showPassword ? 'eye' : 'eye-off'} // ← troca o ícone conforme o estado
      size={20}
      color="#333"
    />
  </TouchableOpacity>
</View>
```

> **Por que `position: absolute`?** Para o ícone flutuar *dentro* do campo sem deslocar o layout. O `View` pai tem `position: relative` para que o `absolute` seja relativo a ele.

## 🔑 Melhoria 2 — Esqueci minha senha

**Arquivo:** `app/(auth)/login.tsx`

### O que foi feito
Adicionado botão de link discreto que envia um e-mail de redefinição de senha via Firebase.

### Import necessário
```tsx
import { ..., sendPasswordResetEmail } from 'firebase/auth';
```

### Função criada
```tsx
const handleForgotPassword = async () => {
  try {
    await sendPasswordResetEmail(auth, email); // usa o e-mail já digitado no campo
    Alert.alert("E-mail enviado!", "Verifique sua caixa de entrada.");
  } catch (error: any) {
    Alert.alert("Erro", "Verifique se o endereço está correto e tente novamente.");
  }
};
```

### Botão no JSX
```tsx
<TouchableOpacity onPress={handleForgotPassword}>
  <Text style={styles.forgotPasswordText}>Esqueci minha senha</Text>
</TouchableOpacity>
```

### Estilo adicionado ao StyleSheet
```tsx
forgotPasswordText: {
  color: '#65b874ff',
  fontFamily: 'Poppins_400Regular',
  fontSize: 13,
  textAlign: 'right',
  textDecorationLine: 'underline',
  margin: 5
}
```

> **Por que não usar loading/ternário aqui?** O estado `loading` existe para o botão de Login, que faz uma operação mais pesada. O "Esqueci minha senha" é um disparo rápido — adicionar um spinner seria excessivo e confuso para o usuário.

## 📧 Melhoria 3 — Templates de E-mail do Firebase

**Onde:** Firebase Console → Authentication → Templates

### O que foi feito
Personalizado o campo **Assunto** dos templates de e-mail para português brasileiro.

**Exemplo aplicado:**
- `Dose Certa — Confirme seu e-mail 💊`
- `Dose Certa — Redefina sua senha`

### ⚠️ Limitação encontrada
O Firebase no plano **Spark (gratuito)** não permite edição do corpo HTML dos e-mails neste projeto. Apenas o assunto e remetente podem ser alterados.

**Para liberar edição completa no futuro:**
- Fazer upgrade para o plano **Blaze (pay-as-you-go)**, ou
- Configurar um servidor SMTP próprio (ex: SendGrid, Mailgun) nas configurações do Firebase

## 🔔 Melhoria 4 — Notificações Locais

**Arquivos:** `services/notifications.ts`, `app/(tabs)/add.tsx`

### O que foi feito
Criado um serviço de notificações locais que agenda um lembrete diário para cada horário de cada remédio cadastrado.

### Serviço criado (`services/notifications.ts`)
```ts
import Constants from 'expo-constants';

// Guard: expo-notifications não funciona no Expo Go SDK 53+
const isExpoGo = Constants.appOwnership === 'expo';

export async function requestNotificationPermission(): Promise<boolean> {
  if (isExpoGo) return false;
  try {
    const Notifications = await import('expo-notifications');
    const { status } = await Notifications.requestPermissionsAsync();
    return status === 'granted';
  } catch {
    return false;
  }
}

export async function scheduleNotification(medicineName: string, time: string): Promise<void> {
  if (isExpoGo) return;
  try {
    const Notifications = await import('expo-notifications');
    const [hours, minutes] = time.split(':').map(Number);
    await Notifications.scheduleNotificationAsync({
      content: {
        title: '💊 Hora do remédio!',
        body: `Está na hora de tomar ${medicineName}`,
      },
      trigger: {
        type: Notifications.SchedulableTriggerInputTypes.DAILY,
        hour: hours,
        minute: minutes,
      },
    });
  } catch {
    // falha silenciosa
  }
}
```

### Integração no `handleSave` (`add.tsx`)
```tsx
// Antes de salvar:
const granted = await requestNotificationPermission();
if (!granted) {
  Alert.alert("Permissão negada", "Ative as notificações para receber lembretes.");
}

// Depois do addDoc, protegido por try/catch próprio:
if (granted) {
  try {
    for (const time of times) {
      await scheduleNotification(name, time);
    }
  } catch {
    console.warn("Notificações não suportadas neste ambiente.");
  }
}
```

> **Por que o import é dinâmico (`await import(...)` dentro da função)?**
> O `expo-notifications` tem efeitos colaterais que rodam no momento do import. No Expo Go SDK 53+, isso causa um crash imediato. Com o import dinâmico + o guard `isExpoGo`, o módulo só é carregado em builds nativos, onde funciona corretamente.

> **Por que as notificações não funcionam no Expo Go?**
> O Expo Go é uma camada JavaScript sobre o sistema operacional. Desde o SDK 53, notificações requerem serviços nativos compilados dentro do app. Para testar notificações, é necessário um **Development Build** (`npx expo run:android`) ou o **APK final**.

## 📳 Melhoria 5 — Feedback Háptico

**Arquivo:** `app/(tabs)/index.tsx`

### O que foi feito
Adicionada vibração física nos botões de check-in de dose, reforçando o feedback sensorial da ação.

### Import
```tsx
import * as Haptics from 'expo-haptics';
```

### Uso nos botões
```tsx
// Botão "Tomado" → vibração forte de sucesso
onPress={() => {
  Haptics.notificationAsync(Haptics.NotificationFeedbackType.Success);
  handleTakeMedication(item, true);
}}

// Botão "Pular dose" → vibração suave de aviso
onPress={() => {
  Haptics.notificationAsync(Haptics.NotificationFeedbackType.Warning);
  handleTakeMedication(item, false);
}}
```

> **Por que `expo-haptics` funciona no Expo Go mas `expo-notifications` não?**
> Vibração é um hardware simples acessível por qualquer app. Notificações requerem um serviço nativo instalado dentro do app, que o Expo Go não inclui desde o SDK 53.

## 👴 Melhoria 6 — Acessibilidade Dinâmica (Fontes Grandes)

**Arquivos:** `app/(tabs)/index.tsx`, `app/(tabs)/explore.tsx`

### O que foi feito
Revisados os cards de remédio para não quebrarem quando o usuário ativa fontes grandes no sistema (Configurações → Acessibilidade).

### Mudanças de estilo (`index.tsx`)

```tsx
// ❌ Antes — largura fixa cortava texto com fontes grandes
cardTitle: {
  width: 180,
}

// ✅ Depois — ocupa o espaço disponível sem transbordar
cardTitle: {
  flex: 1,
}

// ❌ Antes — botão com largura fixa
saveButton: {
  width: 180,
}

// ✅ Depois — flexível, mas com limite máximo
saveButton: {
  flex: 1,
  maxWidth: 180,
}
```

### Mudança no JSX (`index.tsx` e `explore.tsx`)

```tsx
// Evita que nomes longos quebrem o layout — vira "Amoxicili..." em vez de transbordar
<Text
  style={styles.cardTitle}
  numberOfLines={1}
  ellipsizeMode="tail"
>
  {item.name}
</Text>
```

### Correção de overflow no histórico (`explore.tsx`)

```tsx
// ❌ Antes — cardText sem restrição empurrava o status "Tomado/Não tomado" para fora da tela
<View style={styles.cardText}>

// ✅ Depois — flex: 1 garante que o texto ocupa só o espaço disponível
<View style={[styles.cardText, { flex: 1 }]}>

// No StyleSheet:
cardText: {
  flex: 1,  // deixa espaço para o ícone de status à direita
  gap: 5
}
```

> **Regra geral:** Em layouts `flexDirection: 'row'` com múltiplos filhos, evite `width` fixo. Use `flex: 1` no elemento central para que ele ocupe o espaço restante sem empurrar os vizinhos.
